SYSTEMIC_REFACTOR_PROGRAM_V1

Purpose

Establish a Rust/Tauri runtime substrate capable of hosting the existing preserved TypeScript platform without modification.

The purpose of this program is not frontend migration.

The purpose of this program is construction of a new execution and presentation substrate while preserving all existing domain, service, realtime, SSE, workflow, and authentication infrastructure defined under Domain Preservation Law 001.

Constitutional Principle

The current TypeScript domain layer is considered authoritative infrastructure.

The Rust/Tauri runtime is considered a host substrate.

The Rust/Tauri runtime SHALL adapt to the preserved platform.

The preserved platform SHALL NOT adapt to the runtime.

Program Objectives

1. Construct a production-grade Rust/Tauri desktop runtime.

2. Construct a constitutional IPC layer.

3. Construct a desktop infrastructure framework.

4. Construct a visual token system.

5. Construct an observability substrate.

6. Construct an ASCII rendering substrate.

7. Construct a migration-ready application shell.

8. Preserve all Tier 0, Tier 1, and Tier 2 systems.

Program Restrictions

The following systems are constitutionally protected:

* authStore
* supabase.ts
* database.types.ts
* agent-tables.ts
* oversight.ts
* agentService.ts
* useAgentHealth
* useOversightChat
* Realtime Hooks
* Workflow Infrastructure

These systems SHALL NOT be modified.

These systems SHALL NOT be rewritten.

These systems SHALL NOT be migrated.

Adapters MAY be created.

Wrappers MAY be created.

Consumers MAY be replaced.

Core implementations SHALL remain intact.

Completion Condition

The systemic refactor foundation is considered complete when:

* Rust runtime exists
* IPC layer exists
* Desktop infrastructure exists
* Design token system exists
* Observability framework exists
* ASCII framework exists
* Application shell exists

Only after completion may page-level migration contracts be authorized.
